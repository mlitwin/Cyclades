First, while I love abstraction, the explanation in terms of iterated functions was too abstract for me. So, a more concrete case to think about: a nice linked list:

 

Ok it's a singly linked list. You've got a pointer to the first node, and each node has a pointer to the "next" node (or a sentinel for the end of the list). You can start at the first node, and next your way to the end. Unless: some joker has set one of the "next" back to a previous node, in which case you're stuck in an infinte loop. This is the case the Cycle Detection algorithms detect.
